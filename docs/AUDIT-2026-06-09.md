# Repo Audit & Improvement Plan — bitcoinweighin

*Audit date: 2026-06-09. Analysis only; no code modified. All citations verified against the working tree at commit `e654fbc`.*

---

## 1. Executive Summary

Overall health: **B**. This is an unusually disciplined solo project: a fully prerendered SvelteKit site with a resilient daily data pipeline, ~156 unit tests around the core maths, a documented decision log, and clean secret hygiene. What holds it back is that none of that rigour is *enforced* — no CI runs the tests, TypeScript runs non-strict, and the pricing/formatting logic has been copy-pasted into four runtimes, which has already produced one live correctness bug (cocaine priced at $35/g in OG share images vs $30/g everywhere else). Top 3 risks: (1) constant/logic drift between `src/lib`, `functions/`, and `scripts/bot` with no test guarding the seams; (2) zero CI gate — a regression ships silently to a public site with a posting bot; (3) load-bearing files sitting untracked in the working tree, recoverable from nowhere if the machine dies. Top 3 opportunities: (1) a single CI workflow (under an hour's work) converts the existing test suite from decoration into a safety net; (2) a shared pricing/format core eliminates the entire drift class; (3) enabling `strict` while the codebase is this small is cheap and locks in the type quality the code mostly already has.

## 2. Repo Map

**Purpose:** bitcoinweighin.com — visualises BTC purchasing power as physical commodity volumes (gold, silver, Pu-238, cocaine) with a 2013–present history scrubber, daily-updated CC-BY dataset, dynamic OG images, and an X bot. Personal project, late pre-launch/launched maturity (SEO pages and bot live as of June 2026).

**Stack:** SvelteKit 2 + Svelte 5 (runes) + TypeScript + Tailwind 3, `adapter-static` → Cloudflare Pages. Cloudflare Pages Functions for OG images and an LNURL tip proxy. Vitest for unit tests. Node/tsx scripts for the data pipeline and bot; Playwright renders bot cards; Blender (Python-driven) renders sprites. GitHub Actions for the two operational crons.

**Architecture & data flow:**
```
stooq/FRED ──fetch-daily.ts──> data/prices.ndjson ──build-prices-json.ts──> static/prices*.json
                                                  └─build-dataset-artifacts.ts─> static/data/* (CSV/JSON/NDJSON/Parquet)
static/prices.json ──build-time load──> prerendered pages (routes/+page.ts, btc/[commodity], snapshot/[year])
                   ──runtime fetch───> functions/og-image.ts (share cards)
                   ──read───────────> scripts/bot/run.ts ──Playwright card──> X
```

**Key directories:**
- `src/lib/` — core domain: `commodities.ts` (catalogue), `prices.ts`, `volume.ts` (cube maths), `format.ts`, `seo/`, `stores/url.ts` (URL-as-source-of-truth state)
- `src/lib/components/` — 28 Svelte components + TS engines (CubeRenderer, GeigerEngine, glow overlay)
- `src/routes/` — 7 routes, fully prerendered
- `functions/` — Cloudflare Pages Functions (`og-image.ts`, `_lib.ts` acknowledged copy of src/lib bits, LNURL proxy)
- `scripts/` — data pipeline (`fetch-daily`, `build-prices-json`, `build-dataset-artifacts`, holdings sync/scrape), `bot/` (X bot), `blender/` (~35 render scripts incl. iterations)
- `tests/` — 11 Vitest files, ~156 test cases
- `docs/` — SPEC, DECISIONS (exemplary), PROJECT-STATUS (stale), staged handoff docs
- `.github/workflows/` — `daily-update.yml` (02:00 UTC prices), `bot.yml` (5 posts/day) — **data ops only, no test/build CI**

**Surprises found during mapping:** `functions/_lib.ts` openly declares itself a manual copy that must be kept in sync (it has already drifted); `git status` shows untracked load-bearing files (`scripts/bot/make-short.ts`, a handoff doc, two Pu-238 PNGs); `.git` is 119 MB of loose objects that pack to 5.9 MB; the project tsconfig never enables `strict`.

## 3. Audit Report

Severity scale: Critical / High / Medium / Low. Each finding labelled **[fact]** or **[judgment]**.

### 3.1 Correctness & code quality

**Q1 — HIGH [fact]. Cocaine price drift: OG images use $35/g, the site and bot use $30/g.**
`functions/_lib.ts:73` hardcodes `illustrativePricePerUnit: 35`, with a comment claiming "UNODC wholesale tier midpoint". The actual wholesale midpoint is $30/g: `src/lib/illustrative-prices.json` (`tiers.wholesale.pricePerKg: 30000`, consumed at `src/lib/prices.ts:75–77` as `pricePerKg / 1000`), and `scripts/bot/objects.json` (`"usdPerGram": 30`). Every cocaine OG share card understates the quantity by ~14% relative to the page it links to — exactly the "share preview tells the same story as the site" promise `functions/og-image.ts:9–10` makes. This is the proven cost of finding Q2.

**Q2 — HIGH [fact]. Pricing and formatting logic exists in four independent copies.**
(a) `src/lib/prices.ts:66–110` (site); (b) `functions/_lib.ts:107–126` (`computeAmount`, "minimal copy… keep values in sync" per its own header, lines 2–5) plus re-implemented formatters at `_lib.ts:148–255` duplicating `src/lib/format.ts`; (c) `scripts/bot/deltas.ts:45–55` (`gramsPerBtc` reading `objects.json`); (d) verbatim copies of `gramsPerBtc` in `scripts/bot/run.ts:79–84` and `scripts/bot/make-short.ts:70–75`. Price *values* additionally live in three places (`illustrative-prices.json`, `_lib.ts` constants, `objects.json`). No test compares any pair. Q1 demonstrates this isn't theoretical.

**Q3 — HIGH [fact]. TypeScript runs non-strict.**
`tsconfig.json` sets only `allowImportingTsExtensions`/`module`/`moduleResolution`; neither it nor the generated `.svelte-kit/tsconfig.json` contains `"strict": true`, so `noImplicitAny` et al. are off. Symptom: `objs: any` at `scripts/bot/run.ts:79,106,128` and `scripts/bot/make-short.ts:70,108`, `err: any` at `scripts/bot/verify-auth.ts:41` and `scripts/bootstrap.ts:67` — while `deltas.ts:28–40` already defines the proper `ObjectsFile` type the others ignore. There is also no `svelte-check` dependency and no `check` script in `package.json`, so type errors surface nowhere except the editor.

**Q4 — MEDIUM [fact]. Load-bearing files are untracked.**
`git status`: `scripts/bot/make-short.ts` (420 lines, the shorts/video card generator), `docs/handoff/08-gold-history-video.md`, `static/sprites/pu238/cube@2x.png` + `cube-shadow@2x.png` are untracked; `assets/blender/gold_bar_single/`, `gold_coin/` likewise. One disk failure loses them; CI can never use them. (`.wrangler/` is also untracked and should be gitignored instead.)

**Q5 — LOW [judgment]. `src/routes/+page.svelte` (882 lines) concentrates slider maths, dual-mode state, URL hydration, JSON-LD, layout and styles.** It's cohesive and readable, but it's the file most edits touch; the log-slider helpers (lines 53–105) are pure functions that would be trivially testable if extracted.

**Q6 — LOW [fact, accepted convention].** ~25 versioned Blender iteration scripts (`gold_coin_v2..v6`, `smoke_test_v2..v6`) remain in `scripts/blender/`. Comments and `.gitignore` (lines 64–69) show this is deliberate archaeology, not accidental dead code (`.gitignore:61–69`). No action required beyond an owner decision on archiving.

### 3.2 Architecture & design

Healthy overall **[judgment]**: module boundaries in `src/lib` are clean (pure maths in `volume.ts`, data loading in `prices.ts`, URL state isolated in `stores/url.ts` with a well-reasoned hydration gate at `url.ts:30–35`), no circular dependencies observed, and the renderer dispatch (`PhysicalRep.svelte` branching on `renderStyle`) matches the documented design. The one structural defect is the duplication seam described in Q2 — it is an architecture problem (no shared core consumable by site, worker and bot), not a copy-paste accident.

### 3.3 Security

Materially sound for a static site. **Strengths [fact]:** no secrets in git (`git ls-files` shows only `.env.example`; `.env` properly ignored); API keys redacted from logged URLs (`scripts/fetchers.ts:39–41`, `fetchers-massive.ts:44–49`); OG endpoint validates input (commodity whitelisted via object lookup `og-image.ts:297–298`, `btc` coerced to finite positive `og-image.ts:293–294`, `date` only accepted if present in the dataset `og-image.ts:304` — no injection path into the HTML template); JSON-LD escapes `<` (`src/lib/seo/jsonld.ts:15`); workflow permissions scoped to `contents: write` only.

**S1 — LOW [fact]. No CSP / X-Frame-Options / Referrer-Policy on HTML routes.** `static/_headers` sets only caching and `X-Content-Type-Options` on data paths. For a static site with no auth the practical risk is clickjacking/embedding; cheap to add.

**S2 — LOW [fact]. LNURL proxy forwards arbitrary query params upstream with `Access-Control-Allow-Origin: *` and no rate limit** (`functions/.well-known/lnurlp/tips.ts:22–24`). Upstream is a fixed URL so the abuse surface is small; Cloudflare's own limits are the de facto control.

**S3 — LOW [fact]. Holdings scraper trusts a regex first-match from scraped HTML** (`scripts/holdings-sources/bitcointreasuries.ts:43`: first `(\d[\d,]*)\s*BTC` occurrence). Mitigated by the 25% sanity cap (`holdings-sync.ts:27`) and `holdings:check` validation in the workflow. Residual risk: a page redesign that puts a different plausible figure first would pass the cap if within 25%.

**S4 — LOW [judgment].** GitHub Actions pinned by major tag (`actions/checkout@v5`) rather than SHA. Standard practice trade-off; note only.

### 3.4 Testing

**T1 — Strength [fact].** ~156 test cases across 11 files (`grep -c "it(\|test("`), covering the right things: volume/cube-edge canonical values, viewport scaling, glow curves, Geiger engine, pipeline row assembly, render dispatch integration. These assert behaviour, not just execution.

**T2 — HIGH [fact]. No CI ever runs them.** The only workflows are `daily-update.yml` and `bot.yml`. No push/PR workflow runs `vitest`, type-checks, or builds. Cloudflare Pages will surface a *build* failure post-push, but a logic regression (e.g. Q1) ships silently to a live site and a bot that posts publicly five times a day.

**T3 — MEDIUM [fact]. `functions/` has zero test coverage** — and it is precisely the acknowledged-copy code where drift occurs (Q1 lives there). `_lib.ts` is dependency-free pure TS; it is the cheapest possible thing to test.

**T4 — MEDIUM [fact]. Bot logic untested.** `pickSlot` (`scripts/bot/run.ts:152–169`) is exported and pure — clearly written to be tested — but no test exists for slot catch-up selection, dedup, or budget guards that protect a paid posting quota.

**T5 — Accepted gap [fact].** No Playwright visual regression — explicitly deferred per `docs/DECISIONS.md` (2026-04-19); numeric canonical-edge tests stand in. Reasonable for the maturity.

*Honesty note:* the suite could not be executed in this audit's Linux sandbox — `node_modules` is built for macOS arm64 (`@rollup/rollup-linux-arm64-gnu` missing). Counts and quality assessment are from source inspection.

### 3.5 Performance

Healthy; this is a strength. Prerendered HTML with the latest day's prices inlined and the full archive lazy-loaded (`+page.svelte:32–36`), IntersectionObserver lazy-mounting with reserved heights to prevent CLS (`commodities.ts:115–122`), post-build critical-CSS inlining with documented trade-offs (`scripts/inline-critical-css.ts:1–22`), tuned edge cache headers (`static/_headers`), and OG sprite/font caching in the Workers Cache. Two minor notes: **P1 — LOW [fact]** `fetch-daily.ts:74–76, 91–93` re-reads and re-parses the full ~2 MB `prices.ndjson` once per source per date inside the loop — irrelevant at daily-cron scale, mildly wasteful during multi-day catch-up. **P2 — info [fact]** `prices.json` ships in four copies (`static/prices.json`, `static/api/prices.json`, `static/data/prices.json`, plus versioned sets) — deliberate API surface, costs deploy size only.

### 3.6 Dependencies

**D1 — LOW/MEDIUM [fact].** `parquetjs-lite@0.8.7` (used by `build-dataset-artifacts.ts`) is effectively unmaintained upstream. It runs at build time only (no runtime exposure), but a Node major bump could strand it. **D2 — LOW [fact].** `vite@^5.4` + `@sveltejs/vite-plugin-svelte@^4` are a major generation behind; Svelte 5 and Kit 2 are current. **D3 — Strength [fact].** Lockfile committed, `npm ci` in CI, no duplicated heavyweight deps; `playwright` and `@google/model-viewer` are justified by use.

### 3.7 DevEx & operations

**O1 — HIGH [fact]. Nothing enforces anything.** No ESLint/Prettier/Biome config anywhere in the repo, no `svelte-check`, non-strict TS (Q3), no CI gate (T2). Four reinforcing absences = one theme (see Strategy).

**O2 — Strength [fact].** The operational engineering on the data jobs is genuinely good: rebase-and-retry push loops with documented race reasoning (`daily-update.yml:64–83`, `bot.yml:81–99`), a concurrency group serialising bot runs, the all-sources-zero weekday guard (`fetch-daily.ts:292–300`), `health.json` observability with per-source HTTP status and redacted URLs, cross-validation scaffolding that fails soft, and budget/dedup guards before any paid post (`run.ts:200–212`).

**O3 — LOW [fact]. `.git` is 119 MB of loose objects packing to 5.9 MB** (`git count-objects -vH`: size 119.51 MiB, size-pack 5.86 MiB). A `git gc` reclaims ~115 MB. Cosmetic but free.

**O4 — LOW [fact]. No LICENSE file at repo root**, despite `docs/DECISIONS.md` recording "CC-BY-4.0 for the prices dataset. MIT for the code." The dataset licence exists (`static/data/v1.0/LICENSE.txt`); the code licence doesn't. Matters the day the repo flips public.

### 3.8 Documentation

**Doc1 — MEDIUM [fact]. `docs/PROJECT-STATUS.md` claims "Last updated 2 June 2026" but materially contradicts the tree.** It states the Pu-238 cube sprite "does not exist on disk" (it ships, tracked, at `static/sprites/pu238/cube@2x.webp`), and lists OG image generation, the methodology page and the `/data` page as future Phase 4 work (all shipped — `functions/og-image.ts`, `src/routes/methodology/`, `src/routes/data/`). This repo's workflow leans heavily on handoff docs being trustworthy context for agents and future-you; a stale status doc is actively misleading here, more than it would be elsewhere. **Doc2 — Strength [fact].** `DECISIONS.md` and `docs/handoff/` are exemplary — dated, reasoned, never edited retroactively. **Doc3 — LOW [fact].** `README.md` is three lines and doesn't point at `docs/` — trivial to fix, disproportionate onboarding value.

### 3.9 Strengths (what to preserve)

The decision-log culture; the data pipeline's failure-mode thinking (catch-up, forward-fill, health surfacing, soft-fail cross-validation); secret hygiene and URL redaction; input validation on the public worker; the performance posture; real behavioural unit tests around the core maths; gitignore entries with *reasons*; the bot's budget and dedup guards. Recommendations below deliberately fit this culture (small scripts, documented decisions, no heavy frameworks).

## 4. Improvement Strategy

Four themes explain nearly every finding.

**Theme 1 — One source of truth for prices and formatting** (Q1, Q2, T3). The site, the OG worker, and the bot each re-implement "how much commodity does X BTC buy" with independently-maintained constants, and drift has already shipped. *Target state:* one dependency-free TS module (constants + compute + format) consumed by all three runtimes; `functions/` can import plain TS via a relative path without touching the SvelteKit boundary, and the bot already runs tsx. Where a literal copy must remain (e.g. `objects.json` values), a unit test asserts equality across sources. *Principle:* duplication is acceptable only when a test makes silent drift impossible.

**Theme 2 — Make the existing rigour automatic** (T2, Q3, O1). The tests, types, and conventions exist; no machine checks them. *Target state:* a CI workflow on every push runs `svelte-check`/`tsc` (strict), `vitest run`, and `vite build`; `strict: true` in tsconfig; a formatter so diffs stay clean. *Principle:* a solo project needs CI more than a team does — there's no reviewer to catch what you miss at 11pm.

**Theme 3 — The repo must be the whole project** (Q4, Doc1, O3, O4). Untracked load-bearing files and a stale status doc mean the repo no longer fully describes the project. *Target state:* everything load-bearing committed or consciously ignored with a reason; PROJECT-STATUS matches reality; LICENSE present; `.git` packed.

**Theme 4 — Calibrated restraint (what NOT to fix).** Explicitly not recommended at this maturity: Playwright visual regression (already consciously deferred — the canonical-edge tests are a good trade); a monorepo/workspace split for the shared core (a folder + relative imports suffices); CSP with nonces (static site — a simple header is enough); replacing `parquetjs-lite` immediately (build-time only; decide when Node forces it); the Vite 5→7 major (no current pain; bundle with the next Kit-driven upgrade); pruning the Blender script archaeology (it's documented intent, near-zero cost).

**Definition of done — measurable signals:**
- CI badge red on any failing test, type error, or build break (currently: impossible by construction).
- Zero independently-maintained price constants without a cross-source equality test.
- `strict: true` with zero suppressions beyond documented `@ts-expect-error`s.
- `git status` clean; `du -sh .git` under 15 MB; LICENSE at root.
- PROJECT-STATUS.md contains nothing contradicted by the tree.
